# 📝 Notes Manager (Personal Assistant Bot)

Модуль для керування текстовими нотатками з підтримкою інтелектуального тегування, кольорової візуалізації та автоматичного збереження.

## 🚀 Основні команди

### 1. Створення та редагування

* `add-note [текст]` — Створює нову нотатку. Будь-яке слово, що починається з символу `#`, автоматично реєструється як тег.
* *Приклад:* `add-note Підготувати презентацію для #goit на #monday`


* `edit-note [ID] [текст]` — Дописує вказаний текст у кінець існуючої нотатки. Нові теги з тексту також будуть додані до списку.
* *Приклад:* `edit-note 1 Не забути про слайди #it`


### 2. Перегляд та пошук

* `notes` — Виводить повний список усіх нотаток у вигляді структурованої таблиці з ID та кольоровими тегами.

* `find-tag [назва_тегу]` — Фільтрує нотатки та виводить лише ті, що містять вказаний тег.
* *Приклад:* `find-tag goit` або `find-tag #monday`



### 3. Видалення

* `delete-note [ID]` — Видаляє конкретну нотатку за її унікальним номером (ID).
* `clear-notes` — Повне очищення блокнота (потребує підтвердження `y/n`).

---

## 🏷️ Як працюють "Розумні теги"

Ваш помічник автоматично розпізнає ключові слова, що полегшує організацію інформації:

1. **Авто-збір:** Бот сканує ваш текст на наявність символу `#` і самостійно наповнює список тегів.
2. **Кольорова візуалізація:** У загальному списку (`notes`) всі теги підсвічуються **блакитним кольором**, що дозволяє швидко орієнтуватися в записах.
3. **Регістронезалежність:** Теги `#Work`, `#WORK` та `#work` вважаються одним і тим самим тегом.

## ⚙️ Правила та технічні деталі

| Параметр | Опис |
| --- | --- |
| **Ліміт тексту** | Кожна нотатка може містити до **500 символів**. |
| **Система ID** | Кожній нотатці автоматично присвоюється номер (1, 2, 3...), який використовується для редагування та видалення. |
| **Збереження** | Дані автоматично зберігаються у файл `notes.pkl` при виході з програми. |
| **Місце зберігання** | Папка `assistant_bot_data` у вашій домашній директорії користувача (Home Path). |
